# Security Policy

## Supported Versions

Daxerly is a continuously deployed web application. Security fixes are applied to
the latest version running at [daxerly.apoorvdarshan.com](https://daxerly.apoorvdarshan.com)
and the `main` branch of this repository.

| Version        | Supported |
| -------------- | --------- |
| `main` (latest) | ✅        |
| older commits   | ❌        |

## Reporting a Vulnerability

**Please do not open a public issue for security vulnerabilities.**

Instead, report it privately:

- 📧 Email **ad13dtu@gmail.com** with the details, or
- 🔒 Use GitHub's [private vulnerability reporting](https://github.com/apoorvdarshan/daxerly/security/advisories/new).

Please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce (proof of concept if possible)
- Affected URL, endpoint, or file
- Any suggested remediation

## What to Expect

- **Acknowledgement** within 72 hours.
- An assessment and, where valid, a fix timeline.
- Credit for the disclosure if you'd like it, once the issue is resolved.

## Scope

In scope: the hosted app and this repository's source. Please act in good faith:

- Don't access, modify, or delete data that isn't yours.
- Don't run denial-of-service, spam, or automated attacks against the service.
- Don't publicly disclose an issue until it has been resolved.

Thank you for helping keep Daxerly and its users safe.
